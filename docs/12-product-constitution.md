# Constitución de producto

Esta constitución define reglas no negociables para GRIM Analytics. Toda decisión de producto, diseño, IA e ingeniería debe poder justificarse contra ellas.

1. **Toda pantalla debe ayudar a tomar una decisión.** Una vista sin pregunta ejecutiva y siguiente paso debe replantearse.
2. **El impacto económico tiene prioridad visual.** Ingreso, margen, riesgo o tiempo aparecen antes que el detalle operativo.
3. **La IA debe explicar, justificar y recomendar.** Una conclusión sin evidencia y una recomendación sin motivo no son suficientes.
4. **No afirmar capacidades inexistentes.** Los mocks, borradores y acciones preparatorias deben nombrarse como tales.
5. **No usar gráficas que no cambien una decisión.** Una visualización debe responder una pregunta que el texto o una cifra no resuelven mejor.
6. **El usuario no busca problemas; GRIM los prioriza.** Inicio presenta una lista breve y ordenada de lo que merece atención.
7. **Toda recomendación muestra contexto, confianza e impacto.** También declara vigencia, supuestos o datos faltantes cuando sean relevantes.
8. **La simplicidad tiene prioridad sobre la cantidad.** Divulgación progresiva antes que densidad y cobertura exhaustiva.
9. **Cada nueva función debe reducir carga cognitiva.** Si añade decisiones de interfaz sin mejorar la decisión comercial, debe simplificarse.
10. **Si una función no ayuda a decidir mejor, no pertenece a GRIM.** Actividad, decoración y automatización sin resultado verificable quedan fuera.

## Reglas de confianza

- Diferenciar hecho, cálculo, inferencia, pronóstico y recomendación.
- Nunca presentar correlación como causalidad.
- Toda acción sensible requiere confirmación y trazabilidad.
- Una acción mock puede preparar trabajo, pero no confirmar ejecución externa.
- El usuario conserva control, puede corregir contexto y puede inspeccionar evidencia.

## Reglas de atención

- Una señal se eleva cuando combina impacto, urgencia, confianza y capacidad de acción.
- El color no es el único indicador de prioridad.
- La interfaz inicial evita dashboards, tablas extensas y mosaicos de métricas.
- El éxito se mide por decisiones resueltas, no por tiempo de sesión.

## Gobierno

Una excepción requiere registrar: principio afectado, evidencia, alcance, riesgo, responsable y fecha de revisión. Producto, Diseño e Ingeniería comparten la responsabilidad de hacer cumplir esta constitución.
